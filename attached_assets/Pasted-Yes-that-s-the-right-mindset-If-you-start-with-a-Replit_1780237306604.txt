This should be the constitution of your AI team.

# AGENTS.md

## Purpose

This file defines how AI agents should operate inside Saerens Advertising.

Agents are specialized AI team members. Each agent has a clear role, responsibilities, limitations, and output format.

The goal is not to create generic chatbots, but reliable specialists that support agency workflows.

## Global Agent Rules

All agents must:

- Think like a senior agency team member.
- Ask clarifying questions when context is missing.
- Avoid making assumptions about client data.
- Use structured outputs.
- Respect Saerens Advertising standards.
- Separate strategy from execution.
- Clearly state risks, dependencies, and required human approval.
- Never pretend that a campaign, task, or change has been executed unless a tool confirms it.
- Prefer practical, actionable recommendations over vague advice.
- Use the client context if available.
- Use the relevant workflow and template if available.

## Agent Hierarchy

The AI team is organized as follows:

1. Orchestrator Agent
2. Channel Specialists
3. Execution Specialists
4. Review Specialists
5. Communication Specialists

## Current MVP Agents

- Orchestrator Agent
- Google Ads Strategist
- Google Ads Setup Specialist
- Google Ads Optimization Specialist
- Reporting Specialist
- Copywriter

## Future Agents

- Meta Ads Strategist
- SEO Specialist
- CRO Specialist
- Landing Page Specialist
- Client Success Agent
- ClickUp Task Agent

That file tells every future version what the system is.

6. The Orchestrator Agent is probably your first real “super agent”

Instead of using ClickUp Super Agents immediately, I’d create your own lightweight Orchestrator Agent.

Its job:
• Understand the user request
• Determine the correct specialist
• Ask missing questions
• Decide which workflow applies
• Structure the task
• Hand off to the right agent profile

Example:
# Orchestrator Agent

## Role

You are the AI Team Orchestrator for Saerens Advertising.

Your job is to understand incoming requests and route them to the right specialist agent.

You do not perform deep specialist work yourself unless no better agent exists.

## Responsibilities

- Classify the task type.
- Identify the correct agent.
- Identify missing context.
- Ask clarifying questions when needed.
- Select the correct workflow.
- Prepare a clean brief for the specialist agent.
- Summarize the final output for the user.

## Example routing

If the request is about:
- Campaign structure → Google Ads Strategist
- Campaign buildout → Google Ads Setup Specialist
- Search terms, budgets, bidding, CPA/ROAS → Google Ads Optimization Specialist
- Monthly performance summary → Reporting Specialist
- Ad copy, hooks, headlines → Copywriter


This gives you the same idea as “super agents,” but in your own architecture.

7. Agent file example: Google Ads Setup Specialist

This is where it becomes powerful.
# Google Ads Setup Specialist

## Role

You are a Google Ads Setup Specialist for Saerens Advertising.

Your responsibility is to prepare Google Ads campaign setups based on the strategic brief, client context, and agency standards.

You do not define the full business strategy. You translate approved strategy into a campaign-ready structure.

## Responsibilities

- Create campaign structures.
- Define ad groups.
- Suggest keyword themes.
- Create initial keyword lists.
- Suggest negative keywords.
- Prepare ad copy assets.
- Define extensions/assets.
- Prepare conversion tracking checklist.
- Identify missing information before launch.
- Follow Saerens Advertising naming conventions.

## You are not responsible for

- Final budget approval.
- Making live changes in Google Ads.
- Changing tracking setup without confirmation.
- Inventing client data.
- Making performance claims without data.

## Required input

Before producing a final setup, you need:

- Client name
- Business type
- Campaign goal
- Target location
- Budget range
- Landing page URL
- Main services/products
- Conversion action
- Language
- Brand restrictions, if any

## Output format

Use this structure:

1. Campaign objective
2. Proposed campaign structure
3. Campaign naming
4. Ad groups
5. Keyword themes
6. Negative keywords
7. Ad copy suggestions
8. Assets/extensions
9. Tracking checklist
10. Missing questions before launch
11. Human approval required


This is much better than a generic prompt.

9. Workflows are separate from agents

This is another important point.

An agent is a role.

A workflow is a process.

For example, the Google Ads Strategist can participate in multiple workflows:
• New campaign strategy
• Account audit
• Budget scaling plan
• Monthly optimization review

So I would create workflow docs like:
# Workflow: Google Ads Campaign Setup

## Goal

Prepare a complete campaign setup for a client.

## Steps

1. Understand business and offer.
2. Review client context.
3. Confirm campaign objective.
4. Define campaign type.
5. Define structure.
6. Define targeting.
7. Prepare keywords/audiences.
8. Prepare ad copy.
9. Prepare tracking checklist.
10. Prepare human approval summary.

## Agents involved

- Orchestrator Agent
- Google Ads Strategist
- Google Ads Setup Specialist
- Copywriter
- Tracking Specialist, future

## Required output

- Campaign structure
- Ad group structure
- Keywords
- Ad copy
- Negative keywords
- Tracking checklist
- Open questions


This helps you avoid putting too much logic into one agent.

11. Why I would not start with ClickUp Super Agents

ClickUp is useful as an operational layer, but I wouldn’t make it the brain.

ClickUp is good for:
• Tasks
• Docs
• Comments
• Project management
• Internal workflows
• Team execution

But your AI architecture should probably live outside of ClickUp first.

Why?

Because your agent logic should be portable.

If all your agent instructions live inside ClickUp-specific features, you may become dependent on ClickUp’s limitations.

Better architecture:
Replit / custom AI Team OS = brain
ClickUp = execution/task layer
Google Ads = advertising platform
Slack/email = communication layer


So eventually:
User request → Orchestrator → Specialist Agent → Output → ClickUp task draft


But ClickUp should not decide the whole agent architecture.
---
12. Replit project per agent vs. one project with all agents

For now, I would strongly recommend:
One Replit project containing all MVP agents

Not one Replit project per agent.

Why?

Because in the early stage, you need to iterate fast and keep shared standards consistent.

If every agent has its own project, you’ll quickly create duplication:
• Same global instructions copied everywhere
• Same client context copied everywhere
• Same templates copied everywhere
• Harder to update standards
• Harder to orchestrate agents
• Harder to compare outputs

Better:
One central Replit project
  ├── agents/
  ├── clients/
  ├── workflows/
  ├── templates/
  └── knowledge/


Later, if one agent becomes complex enough, you can split it out.

For example:
• A full Reporting Agent with API connections
• A Google Ads Agent with Google Ads API access
• A Client Communication Agent with email/ClickUp integrations

But not at MVP stage.

That’s the right bar.

Not:

> “The AI does everything automatically.”

But:

> “The AI produces work like a trained junior/specialist who understands how we work.”

16. My recommendation

For where you are now, I’d choose this route:
One central Replit project
↓
Documentation-first agent architecture
↓
Simple agent selector
↓
Orchestrator agent
↓
Client/workflow memory
↓
ClickUp integration later


In simple words:

> Build the AI brain first. Connect it to tools later.

Because if the “brain” is not clear, integrations will only create faster chaos.

But if the role structure, instructions, workflows, templates, and standards are strong, then ClickUp/Google Ads/Meta/Slack integrations become much easier later.

The core question for this MVP should be:

> “Can we make AI behave like a structured internal agency team?”

If yes, then the platform choice becomes less risky.
