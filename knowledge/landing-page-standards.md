# Landing Page & Conversion Standards

Baseline standards for how Saerens reviews and recommends landing pages so paid and organic traffic converts. Agents apply these unless a client's context gives a documented reason to deviate. The page is where spend turns into results — a strong page protects the whole investment.

## Principles

- **The page must match the traffic.** Message match between the ad/keyword/query and the page is the first thing to check — broken message match wastes spend.
- **One page, one job.** Each landing page should have a single primary goal and a single primary call to action.
- **Reduce friction, build trust.** Every unnecessary field, distraction, or unanswered objection costs conversions.
- **Recommend, don't deploy.** Agents produce specs, structure, and test hypotheses; a human/dev implements changes.

## Conversion essentials (review every page against these)

1. **Clarity above the fold** — within seconds the visitor knows what's offered, for whom, and what to do next.
2. **Value proposition** — clear, specific, and relevant to the traffic source; benefits over features.
3. **Single primary CTA** — prominent, repeated sensibly, with action-focused wording.
4. **Trust signals** — reviews, ratings, certifications (e.g. Google Partner), guarantees the client can actually back, real proof.
5. **Form / checkout friction** — ask only for what's needed; clear steps; visible progress for multi-step flows.
6. **Objection handling** — answer the obvious "but…" questions (price, risk, time, credibility).

## Recommended structure (adapt to goal)

Hero (value prop + primary CTA) → proof/trust → offer detail/benefits → objection handling → secondary proof → final CTA. Lead-gen pages center the **form/call**; e-commerce pages center the **product and add-to-cart/checkout** path.

## Technical & UX baseline

- **Mobile-first** — most paid social and much search traffic is mobile; the mobile experience is the primary experience.
- **Speed** — slow pages lose conversions; flag Core Web Vitals and heavy assets (coordinate with `knowledge/seo-standards.md`).
- **Accessibility basics** — readable contrast, sensible tap targets, labeled form fields.

## Measurement & testing

- Define the **conversion action** and confirm it's tracked (see `knowledge/analytics-standards.md`) before drawing conclusions.
- Frame improvements as **A/B test hypotheses** with an expected direction; label hypotheses vs data-backed findings.
- Prioritize by **expected impact and effort**; never promise a specific uplift.

## Output structure (Landing Page / Web Design Specialist)

Summary → message match → conversion review → structure recommendation → copy & layout notes → technical/UX findings → prioritized recommendations (impact/effort) → test hypotheses → open questions/missing data → human approval required.
