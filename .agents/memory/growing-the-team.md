For the "AI Team System Map" / Saerens Advertising agent documentation:

**Rule:** prefer **deepening an existing agent** (give it an extra output mode or
sub-specialty plus the matching build-time skills) over creating a new agent.
Create a **new agent only when the capability or its boundary is genuinely
distinct** — a different domain, a different kind of output, or a boundary that
would blur an existing agent's clear role if folded in.

**Why:** the user (agency owner) explicitly does not want employee/agent sprawl;
a focused team is easier to reason about and route between. Encoded in
`AGENTS.md` under "Growing the team (new agent vs deeper specialty)".

**How to apply:** when asked to add capability, first ask "is this a deeper branch
of an existing agent?" before proposing a new agent file. Present new-vs-deepen as
an explicit choice.
